# 年度相册网站项目计划

## 目标

建设一个年度相册网站，支持用户上传照片，系统按年份组织相册；每个年度相册使用上传者的行书姓名/署名作为封面视觉元素，并根据该年度照片的整体摄影风格生成或匹配相册视觉风格。

## 假设

- 第一版以私人与家庭/个人作品集场景为主，不先做大型社交平台。
- “上传照片人的行书”表示封面上展示上传者姓名或署名的行书字体/书法风格，而不是必须训练个人手写模型。
- “相册风格贴近拍照风格”第一版使用照片元数据、色彩、明暗、主题标签和人工可调模板来实现，不直接训练专属视觉模型。
- 需要先保护照片隐私和原图资产，避免公开桶、可枚举链接和未授权访问。
- 网站视觉要有摄影作品集级别的高级感：大图优先、留白充分、排版克制、动效服务于浏览照片，不做廉价装饰。
- 年度相册详情页要采用接近真实相册的翻页交互，并保留“首页、上一页、下一页”三个显式按钮。
- 用户可以为某个年度相册生成分享链接，访客无需登录即可在授权范围内观看。

## 技术栈决策

### 推荐栈

- 前端/全栈框架：Next.js + TypeScript
- UI：Tailwind CSS + shadcn/ui + Framer Motion
- 相册翻页：react-pageflip，必要时补充 CSS 3D 阴影、纸张厚度和翻页音效开关
- 数据库：PostgreSQL
- ORM：Prisma
- 对象存储：S3 兼容存储，开发期可用 Cloudflare R2、AWS S3 或 MinIO
- 图片处理：Sharp
- 认证：Auth.js 或 Clerk
- 后台任务：BullMQ + Redis，或托管平台队列
- AI/视觉分析：OpenAI/Google Vision/本地 CLIP 任选其一，先做可替换接口
- 部署：Vercel/Netlify 部署前端与 API，数据库与存储独立托管

### 第一版范围

- 上传照片
- 按上传照片的年份归档，优先读取 EXIF 拍摄年份，缺失时回退到上传时间
- 年度相册列表与详情页
- 自动生成年度封面
- 支持上传者姓名/署名并以行书风格展示
- 提取年度照片风格摘要：色调、明暗、主题、构图倾向
- 根据风格摘要选择相册主题模板
- 年度相册真实翻页浏览
- 年度相册分享链接生成、复制、撤销

### MVP 执行路线

1. 定 MVP：上传照片、按照片年份归档、年度相册、行书封面、风格模板。
2. 做数据模型：User、Photo、AlbumYear、StyleProfile、CoverAsset。
3. 实现上传和图片处理：原图存储、缩略图、EXIF 年份识别。
4. 做风格分析：主色、亮度、饱和度、主题标签，映射到相册模板。
5. 补体验和上线：响应式、上传进度、私密访问、部署配置。

## 阶段计划

### 阶段 1：需求收敛

状态：complete

任务：
- 明确用户角色、隐私边界、照片规模和是否需要多人协作。
- 明确行书封面是字体渲染、AI 生成字形，还是用户上传手写签名。
- 明确照片风格贴近的实现方式：模板匹配、AI 分析、人工选择或混合。

验证：
- 形成 MVP 功能清单和非目标清单。

### 阶段 2：原型与信息架构

状态：pending

任务：
- 设计首页、年度相册页、照片上传页、照片详情页。
- 定义年度相册封面结构：年份、行书署名、代表照片、风格色板。
- 定义高级感视觉规范：版心、留白、字体层级、照片比例、深浅主题、动效节奏。
- 定义相册阅读器布局：真实书本比例、硬封面、照片跨页、页码、首页/上一页/下一页按钮。
- 定义上传流程和失败状态。

验证：
- 完成低保真页面结构、核心用户路径和年度相册阅读器交互稿。

### 阶段 3：数据模型与存储

状态：pending

任务：
- 设计 User、Photo、AlbumYear、StyleProfile、CoverAsset 等表。
- 分享链接作为后续访问控制模型扩展，核心 MVP 数据模型先围绕 User、Photo、AlbumYear、StyleProfile、CoverAsset 收敛。
- 设计原图、缩略图、封面图的对象存储路径。
- 定义 EXIF、AI 标签、风格摘要的存储字段。

验证：
- Prisma schema 能表达上传、归档、封面生成和风格分析流程。

### 阶段 4：上传与年度归档

状态：pending

任务：
- 实现登录态上传。
- 上传原图到对象存储。
- 用 Sharp 生成缩略图。
- 读取 EXIF DateTimeOriginal 得到照片年份，缺失时使用上传时间。
- 自动归入对应年份相册。

验证：
- 上传多张不同年份照片后，年度相册正确分组。

### 阶段 5：风格分析与封面生成

状态：pending

任务：
- 提取年度照片的主色、亮度、饱和度和方向。
- 可选接入视觉模型生成主题标签，例如人像、旅行、街拍、胶片感、夜景。
- 将分析结果映射到相册主题模板。
- 用行书字体/签名图层叠加生成封面。

验证：
- 同一年度照片能生成稳定的 StyleProfile 和封面预览。

### 阶段 6：相册阅读器与分享链接

状态：pending

任务：
- 使用 react-pageflip 实现年度相册翻页阅读器。
- 保留固定位置的首页、上一页、下一页按钮，并支持键盘方向键。
- 针对移动端切换为单页翻阅，桌面端优先双页摊开。
- 实现分享链接生成、复制、撤销、过期时间设置。
- 访客通过分享链接只能读取派生展示图，不能访问原始私有对象路径。

验证：
- 年度相册可以从封面翻到任意照片页，首页/上一页/下一页按钮行为正确。
- 撤销或过期后的分享链接不可访问。

### 阶段 7：体验打磨与上线

状态：pending

任务：
- 做响应式适配。
- 加上传进度、重试和错误提示。
- 加访问控制和私密分享链接。
- 配置生产环境存储、数据库、缓存和域名。

验证：
- 桌面和移动端核心流程可用；照片非公开资源不可被未授权访问。

## 风险

- AI 生成“个人行书”可能涉及版权、肖似和用户授权问题，MVP 应先采用授权字体或用户上传签名图。
- 大图上传和批量处理容易拖慢请求，图片处理应进入后台任务。
- 仅靠 AI 判断拍摄风格可能不稳定，应允许用户手动切换主题模板。
- EXIF 可能缺失或被社交软件清除，需要上传时间兜底和手动改年份。
- 真实翻页动画如果过重，会影响移动端性能；需要为低端设备提供降级动画。
- 分享链接如果设计成永久公开链接，会带来隐私风险；默认应可撤销、可过期、不可枚举。
