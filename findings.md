# 年度相册网站规划发现

## 技术判断

- 项目本质是图片资产管理 + 年度归档 + 视觉封面生成，优先选择全栈 TypeScript 可以减少前后端模型割裂。
- 原图不应存进数据库，应放对象存储；数据库只保存元数据、访问权限和派生资源路径。
- 年份归档应优先依赖 EXIF 拍摄时间，缺失时回退到上传时间，并提供手动修正。
- 相册风格可以从主色、亮度、饱和度、构图比例和视觉标签中抽象为 StyleProfile，再映射到主题模板。
- 行书封面第一版不要训练个性化模型；更稳妥的方案是授权行书字体、SVG/PNG 签名上传，或后续接入 AI 生成但要求用户确认授权。

## 推荐 MVP

- 单用户或小团队登录。
- 批量上传照片。
- 自动按照片年份创建相册，优先使用 EXIF 拍摄年份，缺失时回退到上传时间。
- 年度相册封面：代表照片背景 + 行书署名 + 年份 + 风格色板。
- 风格模板：清透旅行、胶片日常、低调夜景、明亮人像、自然风光。
- 用户可手动切换封面模板和代表照片。
- 年度相册阅读器使用真实翻页交互，保留首页、上一页、下一页按钮。
- 年度相册支持生成可复制、可撤销、可过期的分享链接。

## MVP 实施顺序

1. 上传照片、按照片年份归档、年度相册、行书封面、风格模板。
2. User、Photo、AlbumYear、StyleProfile、CoverAsset 数据模型。
3. 原图存储、缩略图、EXIF 年份识别。
4. 主色、亮度、饱和度、主题标签，映射到相册模板。
5. 响应式、上传进度、私密访问、部署配置。

## 视觉参考方向

- 参考 Awwwards 摄影类网站的共同方向：照片内容本身是主角，页面设计的任务是更好地展示作品，而不是用装饰抢走注意力。
- 首页采用沉浸式年度封面墙或横向年份时间轴，不做营销式大段说明。
- 年度相册页采用接近实体画册的比例、纸张阴影、硬封面质感和克制的按钮组。
- UI 颜色从照片风格摘要中提取，但页面基础层保持中性，避免每个主题都变成单一色系。

## 翻页实现判断

- `react-pageflip` 支持 React 组件化页面、硬封面、阴影、移动端滚动支持和翻页事件，适合作为第一版真实翻页基础。
- 需要额外补足产品级体验：首页按钮、上一页、下一页、页码、键盘操作、移动端单页模式、低性能设备降级。

## Tailwind 集成判断

- 当前 Tailwind v4 官方建议通过 PostCSS 集成到 Next.js：安装 `tailwindcss`、`@tailwindcss/postcss`、`postcss`，在 `postcss.config.mjs` 中配置 `@tailwindcss/postcss`，并在全局 CSS 中 `@import "tailwindcss"`。
- Next.js 官方 Tailwind 指南说明 v3 配置方式已不是最新版首选；最新版 Tailwind 4 设置应参考 Tailwind 官方安装说明。

## 分享链接判断

- 分享链接不应直接暴露对象存储原始 URL。
- 建议使用随机 token，例如 `/share/:token`，服务端校验 token 状态后返回相册展示数据。
- ShareLink 至少包含：所属相册、token hash、过期时间、撤销状态、访问次数、创建者。
- 可选增强：访问密码、水印、禁止下载原图、访问日志。

## 暂不纳入第一版

- 训练上传者专属书法模型。
- 公共社交流。
- 复杂多人权限体系。
- 自动人脸识别和人物聚类。
- 大规模视频管理。
- 第一版不做不可撤销的永久公开相册。
